IPv4 Addressing Fundamentals

Understanding Internet Protocol Version 4

What is an IPv4 Address?

An IPv4 address is a 32-bit unique identifier that identifies a device on a network. It's written in dotted decimal notation with 4 octets (each 0-255).

  • ‚Ä¢ Range: 0-255

Address Format

  • ‚Ä¢ Notation: 192.168.1.100
  • ‚Ä¢ Decimal (dotted): User-friendly
  • ‚Ä¢ Binary: 32 continuous bits
  • ‚Ä¢ Also written: 192.168.1.0/24 (CIDR)

üß¨ Network & Host Portions

Every IP address is divided into two parts:

üî¥ Network Portion

  • ‚Ä¢ Identifies the network
  • ‚Ä¢ Same for all devices on that network
  • ‚Ä¢ Determined by subnet mask
  • ‚Ä¢ Example: 192.168.1 (first 3 octets)

üü¢ Host Portion

  • ‚Ä¢ Identifies specific device
  • ‚Ä¢ Unique on that network
  • ‚Ä¢ Also determined by subnet mask
  • ‚Ä¢ Example: 100 (last octet)

üìã Subnet Masks

A subnet mask determines which part of the IP is the network and which is the host:

IP Address: 192.168.1.100
Subnet Mask: 255.255.255.0
Network: 192.168.1.0
Host Range: 192.168.1.1 to 192.168.1.254
Broadcast: 192.168.1.255

The mask "255" means "this part identifies the network". The "0" means "this part identifies the host".

Network Classes & Address Ranges

Classful Addressing (Classes A-E)

What are Network Classes?

IPv4 addresses are divided into 5 classes (A-E) based on the first octet value. This determines network size and usage.

CLASS A

Large Networks

Range

First Octet: 1-126
Range: 1.0.0.0 ‚Üí 126.255.255.255
RESERVED: 127.x.x.x (Loopback)

Characteristics

  • ‚Ä¢ Default Mask: 255.0.0.0 (/8)
  • ‚Ä¢ Networks: 126
  • ‚Ä¢ Hosts per network: 16,777,214
  • ‚Ä¢ Format: Network.Host.Host.Host
CLASS B

Medium Networks

Range

First Octet: 128-191
Range: 128.0.0.0 ‚Üí 191.255.255.255

Characteristics

  • ‚Ä¢ Default Mask: 255.255.0.0 (/16)
  • ‚Ä¢ Networks: 16,384
  • ‚Ä¢ Hosts per network: 65,534
  • ‚Ä¢ Format: Network.Network.Host.Host
CLASS C

Small Networks (Most Common)

Range

First Octet: 192-223
Range: 192.0.0.0 ‚Üí 223.255.255.255

Characteristics

  • ‚Ä¢ Default Mask: 255.255.255.0 (/24)
  • ‚Ä¢ Networks: 2,097,152
  • ‚Ä¢ Hosts per network: 254
  • ‚Ä¢ Format: Network.Network.Network.Host
CLASS D

Multicast

Range

First Octet: 224-239
Range: 224.0.0.0 ‚Üí 239.255.255.255

Usage

  • ‚Ä¢ Reserved for multicast
  • ‚Ä¢ One-to-many transmission
  • ‚Ä¢ Video streaming, etc.
CLASS E

Reserved/Experimental

Range

First Octet: 240-255
Range: 240.0.0.0 ‚Üí 255.255.255.255

Usage

  • ‚Ä¢ Reserved for research
  • ‚Ä¢ Not for general use
  • ‚Ä¢ Future expansion

Subnetting: Dividing Networks

The Art and Science of Network Segmentation

What is Subnetting?

Subnetting is dividing a large network into smaller networks (subnets). This saves IP addresses and improves network organization and security.

üéØ Why Subnet?

Efficiency

  • ‚Ä¢ Use IP addresses wisely
  • ‚Ä¢ No wasted addresses
  • ‚Ä¢ Organize by department/function

Security & Management

  • ‚Ä¢ Isolate network traffic
  • ‚Ä¢ Easier troubleshooting
  • ‚Ä¢ Better access control

üìê Subnetting Formula

Given a network, calculate subnets using these formulas:

Number of Subnets: 2^S
where S = number of subnet bits borrowed

Hosts per Subnet: 2^H - 2
where H = number of host bits remaining
(Subtract 2: one for network, one for broadcast)

Example Calculation

Given: 192.168.1.0/24 network, need 4 subnets

Step 1: Calculate bits needed: 2¬≤ = 4 subnets (borrow 2 bits)

Step 2: New prefix: /24 + 2 = /26

Step 3: New mask: 255.255.255.192

Step 4: Hosts per subnet: 2^6 - 2 = 62 hosts

Subnets Created:

192.168.1.0/26 (hosts: 1-62)
192.168.1.64/26 (hosts: 65-126)
192.168.1.128/26 (hosts: 129-190)
192.168.1.192/26 (hosts: 193-254)
